# Project Constitution: AI Shared Memory Browser Extension

## Project Vision

An intuitive cross-browser extension and supporting SaaS platform that solves the context fragmentation problem for multi-LLM users. By automatically capturing chat histories from popular web-based AI clients, condensing them into optimized system summaries, and injecting them seamlessly into target interfaces, the platform enables users to switch tools frictionlessly without repeating context.

## Core Principles

- **Context Optimization over Raw Copying**: Summaries generated by the platform must be highly instruction-optimized and dense, protecting the user's token budget and rate limits on destination platforms.
- **Deterministic UI Extraction**: DOM parsing and selector extraction must rely on robust, deterministic patterns to eliminate hallucinations or erroneous chat history capture.
- **Privacy and Data Sovereignty**: Conversation history is highly sensitive. The storage, processing, and transmission of user contexts must be treated as highly privileged data with minimal retention windows.
- **Frictionless Workflow**: The user interface must proactively assist the user with automated or single-click injections, requiring zero configuration or complex manual adjustments during tab transitions.

## Technology Constraints

- **Extension Frontend**: MUST be built using **TypeScript** and **React** to ensure strong typing, scalability, and easy UI debugging.
- **Backend Service**: MUST use **Python** with **FastAPI** to enable robust, reliable integration with AI orchestrations, structured data generation tools, and parsing utilities.
- **Database & Vector Store**: MUST use **PostgreSQL with the pgvector extension** to keep relational user accounts, session data, and vector memory fully unified, transactional, and scalable in a single engine.

## Architecture Constraints

- **Containerized Services**: All backend services, processing workers, and API endpoints MUST run in **Docker containers** to ensure reliable, scalable, and environment-agnostic deployments.
- **Domain-Isolated Content Scripts**: Browser extension scrapers and injectors MUST be decoupled and isolated into modular domain-specific scripts (e.g., separate modules for Claude, ChatGPT, Gemini) to prevent global namespace pollution and ease updates.
- **Strict Schema Enforcement**: The integration between the extension client and the backend APIs MUST enforce strict JSON/OpenAPI schema-based validation on all requests and responses.

## Testing Approaches

- **Strict End-to-End (E2E) UI Testing**: MUST use **Playwright** to run automated integration tests simulating DOM scraping and text injection across all targeted LLM web interfaces.
- **Code Coverage Target**: The codebase MUST maintain a minimum of **80% automated test coverage** on core parsing patterns, context transformation algorithms, and API sync endpoints.
- **Frequent Interface Smoke Tests & Alerting**: Continuous Integration (CI) pipelines must execute automated interface checks against target portals (ChatGPT, Claude, Gemini) at least every 6 hours (four times daily) coupled with an immediate developer alerting mechanism when a target interface selector fails.

## Coding Standards

- **Strict TypeScript Rulebase**: The extension project configuration must enforce `strict: true` and disallow the use of implicit `any` types.
- **Python Code Quality**: Python code must conform to PEP 8 standards, enforced via `Ruff` or `Black`, with mandatory type hinting checked by `mypy`.
- **Pure Function Scrapers**: Scraper logic extracting data from the DOM must be written as pure, side-effect-free functions that accept document fragments and return structured JSON schemas.

## Security Constraints

- **Secure Extension Storage**: Sensitive session tokens, credentials, and configuration states MUST be stored in the browser's isolated `chrome.storage.local` sandbox with strict permission scopes.
- **Transport Security**: All API traffic between the extension client and the backend MUST be encrypted via TLS 1.3.
- **Sanitized Prompts**: All retrieved context snippets MUST go through injection-prevention sanitization before being inserted into any prompt template or target chat interface.

## Performance Targets

- **Zero-Lag Tab Injection**: The extension must detect and present context injection prompts within **300ms** of a user switching active tabs.
- **Optimized Latency for Summarization**: Summarization operations running on the backend must deliver completed summaries in under **1.5 seconds** (measured as time-to-first-token [TTFT] using stream processing, or overall latency for non-streamed requests by using highly-optimized summarization models and server-side caching).

## Integration Points

- **Target AI Environments**: Web clients of ChatGPT (`chatgpt.com`), Claude (`claude.ai`), Gemini (`gemini.google.com`), and local host environments (such as Ollama or LM Studio UI wrappers).
- **Core AI Providers**: OpenAI and Anthropic API endpoints for generating compressed user summaries.
